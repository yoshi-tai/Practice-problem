# 問題自動作成_プロンプト

## 目的

Practice-problemフォルダ内の各マークダウンファイルを基に、テンプレートと同一構成のHTML・JavaScript・CSSを自動生成・更新してください。

---

# 注意事項

## 1. 実行条件

- フォルダ名とマークダウンファイル名（拡張子を除く）が**完全一致**している場合のみ、本プロンプトを実行してください。
- 一致しない場合は処理を開始せず、その旨を報告してください。

---

## 2. 改善提案

本プロンプトは継続利用することを前提としています。

処理完了後に、品質向上につながる内容があれば最後に「改善提案」としてまとめてください。

改善提案は以下の3項目に分類してください。

### 追加

追加すると品質向上につながる内容

### 更新

現在の記載内容を更新した方がよい内容

### 修正

誤解を招く表現や修正した方がよい内容

それぞれについて、

- 対象箇所
- 理由

を箇条書きで簡潔に記載してください。

---

## 3. Microsoft Copilot向け

本プロンプトは **VS Code の Microsoft Copilot** で実行することを前提とします。

そのため、

- 命令形式で記載すること
- 曖昧な表現を避けること
- 推測ではなく明確な指示を優先すること

---

# プロンプト

## ① テンプレート

「疾病1」フォルダ内の

- one.html
- one.js
- one.css

を完成テンプレートとしてください。

すべての問題はこの構成を基準に作成してください。

以下については原則としてテンプレートと同一構成を維持してください。

- HTML構造
- JavaScript処理
- CSSクラス
- id
- data属性
- イベント処理
- 画面表示
- 処理フロー

必要最小限以外の変更は禁止します。

---

## ② 命名規則

以下はテンプレートを踏襲してください。

- HTML構造
- JavaScriptクラス
- CSSクラス
- id
- data属性
- イベント処理
- 関数構成
- 命名規則

---

## ③ 画面構成

以下は one.html を基準にしてください。

- 画面レイアウト
- 表示順
- ボタン配置
- コンテンツ構成

---

## ④ 問題データ

すべての問題について以下を実装してください。

- 答え
- 答えの解説
- 選択肢ごとの解説

Markdownに記載がない場合でも、正しい内容を補完してください。

---

## ⑤ Markdownの解析

Markdownファイル内の

```
##
```

から始まる見出しを1問として認識してください。

以下のような表記ゆれがあっても1問として扱ってください。

- 問題○
- 問題○○
- 第○回
- 第○○回
- 演習○

各問題ごとにHTMLファイルを1つ生成してください。

---

## ⑥ 内容補正

Markdown内に

- 誤記
- 不足
- 不正確な内容

がある場合は、正しい内容へ補正した上で

- HTML
- JavaScript
- CSS

を作成してください。

---

## ⑦ 文章問題

文章問題の場合は、

問題文の上に

```
題名
```

を表示してください。

画面には以下を表示してください。

- 題名
- 問題文
- 答え
- 答えの解説
- 選択肢の解説

1問につきHTMLファイルを1つ作成してください。

---

## ⑧ 不明点

以下に該当する場合は、処理を開始しないでください。

- 不明点がある
- 判断できない内容がある
- テンプレートとの差異がある

必ず箇条書きで質問してください。

推測による実装は禁止します。

---

## ⑨ HTML生成

対象Markdown内の全問題についてHTMLを生成してください。

JavaScript・CSSは

- one.js
- one.css

を基準として適用してください。

---

## ⑩ 更新対象

処理対象は

```
Practice-problem
```

フォルダ配下のすべての対象フォルダです。

対象フォルダ内のMarkdownから

- HTML
- JavaScript
- CSS

を更新してください。

---

## ⑪ index画面

Markdownファイルごとに

```
index.html
```

へ遷移するボタンを追加してください。

ボタン名は

```
Markdownファイル名（拡張子を除く）
```

を使用してください。

デザイン・イベント処理は

- index.html
- index.js
- index.css

と同一方式で実装してください。

---

## ⑫ 出力対象

作成・更新するファイルは以下のみとしてください。

- index.html
- index.js
- index.css
- 各問題フォルダ内のHTML
- 必要に応じたJavaScript
- 必要に応じたCSS

それ以外のファイルは変更しないでください。

---

## ⑬ エラー時の動作

途中で

- エラー
- 不足情報

が見つかった場合は処理を中断してください。

不足内容を一覧で報告してください。

不足情報が解消されるまでファイル生成を開始しないでください。

---

## ⑭ 実装優先順位

以下の順番で実装してください。

1. one.html
2. one.js
3. one.css
4. index.html
5. index.js
6. index.css

テンプレートとの差異は最小限にしてください。
