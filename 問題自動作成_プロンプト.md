# 問題自動作成_プロンプト

## 1.目的

Practice-problemフォルダ内の各Markdownファイルを基に、既存の問題用テンプレート構成に沿ってHTML・JavaScriptを自動生成・更新してください。

---

# 注意事項

## 1. 実行条件

- フォルダ名とMarkdownファイル名（拡張子を除く）が完全一致している場合のみ、このプロンプトを実行してください。
- 一致しない場合は処理を開始せず、その旨を報告してください。
- 実行時は、既存のテンプレートとして [疾病1_1回目](疾病1_1回目) 配下の実ファイルを優先してください。

---

## 2. 重要な実装ルール

- 生成対象は、対象フォルダ配下のMarkdownファイルから抽出した問題ごとに1ページのHTMLを生成すること。ただし、既存の `one.html` を問題1としてそのまま利用し、Markdownの問題2以降を自動生成すること。
- 生成するHTMLでは、アコーディオン内の内容を以下のように配置してください。
  - 【答えの解説】には、Markdownの「正解の説明」または「答えの根拠」に相当する内容を表示すること。
  - 【選択肢の解説】には、Markdownの各選択肢ごとの解説を表示すること。
  - 選択肢の横に解説を表示しないこと。
- 上記のルールは、対象フォルダ内のすべての問題ページに一括適用してください。
- 既存のテンプレートとして、以下の実ファイル構成を使用してください。
  - [疾病1_1回目/one.html](疾病1_1回目/one.html)
  - [疾病1_1回目/shippei1.js](疾病1_1回目/shippei1.js)
  - [疾病1_1回目/answer.js](疾病1_1回目/answer.js)
  - [style.css](style.css)
  - [page_move/page_move.js](page_move/page_move.js)
  - [page_move/page_move.css](page_move/page_move.css)
- 既存のファイル名が存在する場合は、その命名規則を優先してください。
- 具体的には、問題ページは `one.html` を問題1としてテンプレートとして扱い、Markdownの問題2以降を `two.html`、`three.html` などとして生成してください。
- 既存の構造にある `one.html` などの内容は、必要最小限の変更で維持してください。

---

## 3. Microsoft Copilot向け

本プロンプトは VS Code の Microsoft Copilot で実行することを前提とします。

そのため、

- 命令形式で記載すること
- 曖昧な表現を避けること
- 推測ではなく、既存のファイル構成に沿った明確な指示を優先すること

---

# 実行手順

## ① テンプレート

対象フォルダ内の既存HTML・JavaScript・CSSを基準に、以下の構成で生成してください。

- HTML構造
- JavaScript処理
- CSSクラス
- id
- data属性
- イベント処理
- 画面表示
- 処理フロー

必要最小限以外の変更は禁止します。

---

## ② 命名規則

以下は既存テンプレートを踏襲してください。

- HTML構造
- JavaScriptクラス
- CSSクラス
- id
- data属性
- イベント処理
- 関数構成
- 命名規則

---

## ③ 画面構成

各問題ページは、既存のテンプレートと同じ画面構成で作成してください。

- 問題番号表示
- 答えボタン
- 問題文
- 選択肢
- OKボタン
- 次へ・戻るボタン
- 答え・解説・選択肢解説モーダル

---

## ④ 問題データ

すべての問題について、以下を実装してください。

- 答え
- 答えの解説
- 選択肢ごとの解説

Markdownに記載がない場合でも、正しい内容を補完してください。

---

## ⑤ Markdownの解析

Markdownファイル内の `##` から始まる見出しを1問として認識してください。

以下のような表記ゆれがあっても1問として扱ってください。

- 問題○
- 問題○○
- 第○回
- 第○○回
- 演習○
- 補足問題○

各問題ごとにHTMLファイルを1つ生成してください。

---

## ⑥ 内容補正

Markdown内に

- 誤記
- 不足
- 不正確な内容

がある場合は、正しい内容へ補正した上でHTML・JavaScript・CSSを作成してください。

---

## ⑦ 不明点

以下に該当する場合は、処理を開始しないでください。

- 不明点がある
- 判断できない内容がある
- テンプレートとの差異がある

その場合は、箇条書きで質問してください。

推測による実装は避けてください。

---

## ⑧ 実行コマンド

このプロンプトの実行は、次のコマンドで行ってください。

```bash
python generate_problems_js.py
```

---

## ⑨ 更新対象

処理対象はPractice-problemフォルダ配下の各対象フォルダです。

対象フォルダ内のMarkdownから、

- HTML
- JavaScript
- CSS

を更新してください。

---

## ⑩ 出力対象

作成・更新するファイルは以下のみとしてください。

- 各問題フォルダ内のHTML
- 各問題フォルダ内の `answer.js`
- 各問題フォルダ内の `shippei1.js`

それ以外のファイルは変更しないでください。

---

## ⑪ 質問・改善提案

実行後に、以下があれば質問または改善提案としてまとめてください。

- 生成ルールに関する曖昧さ
- 既存テンプレートとの差分
- 追加した方がよい改善点

特に、次の点は確認してください。

- 既存テンプレートで使われているファイル名
- 画面レイアウトや遷移ルール
- 問題の見出しパターン

---

## 12. 反映内容

今回の実行では、既存のテンプレート構成に合わせて、以下を反映しました。

- Markdownから問題数・解答・選択肢・解説を抽出する処理
- 各問題ごとにHTMLページを生成する処理
- 各フォルダ用の `answer.js` と `shippei1.js` を生成する処理

この内容に矛盾や追加したいルールがあれば、次のように質問してください。

- 既存テンプレートのファイル名を別名にしたい
- 1問ごとのページ名を別形式にしたい
- 問題本文の表示方法を変更したい
